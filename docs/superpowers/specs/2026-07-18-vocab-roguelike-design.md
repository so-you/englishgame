# 英语爬塔（English Roguelike）设计文档 v2

日期：2026-07-18（v2 按评审意见修订）
状态：**待修订 → 修订版，待复审**

## 1. 产品定位

一款英语学习的卡牌 Roguelike 游戏，玩法框架参考杀戮尖塔（Slay the Spire）。

- **目标用户**：中文英语学习者（产品化方向）
- **平台**：Web 浏览器（验证期），后续可包壳
- **定位**：**游戏为体，学习为主题**。先保证好玩、留得住人；学习效果通过机制设计内生获得
- **学习内容**：不限于单词。底层模型抽象为「学习项目」（词汇 + 语法），内容包分为**初中 / 高中 / 大学 / 自定义**四类。MVP 只发布一个内置词库，但数据模型不写死任何级别
- **核心信条**：「变强」和「学会」是同一件事，不是交易

## 2. 数据模型（先于玩法定稿）

### 2.1 学习项目抽象

```ts
type LearningItem = VocabularyItem | GrammarItem

interface VocabularyItem {
  id: string
  type: 'vocab'
  word: string
  phonetic: string
  pos: string                    // 词性
  definitions: string[]          // 中文释义（支持多义词）
  example: string
  exampleZh: string
  roots?: string                 // 词根词缀
  levelTag?: LevelTag            // 人工分级标签（内置包必填）
  frequencyRank?: number         // COCA 词频排名（自定义词表兜底排序用）
  difficultyScore?: number       // 算法兜底难度分
  source: 'builtin' | 'import' | 'llm-enriched'
}

interface GrammarItem {
  id: string
  type: 'grammar'
  grammarPoint: string           // 语法点，如「一般过去时」
  questionType: 'fill-blank'     // 选词填空
              | 'reorder'        // 句子排序
              | 'error-correction' // 改错
              | 'transformation' // 句型转换
  prompt: string                 // 题干
  options?: string[]
  answer: string | string[]
  explanation: string
  levelTag?: LevelTag
  source: 'builtin' | 'import' | 'llm-enriched'
}

type LevelTag = 'junior' | 'senior' | 'cet4' | 'cet6' | 'custom'

interface ContentPack {
  id: string
  name: string
  level: 'junior' | 'senior' | 'university' | 'custom'
  items: LearningItem[]
}
```

### 2.2 SRS 状态与回忆日志

```ts
interface SrsState {
  itemId: string
  mastery: 0 | 1 | 2 | 3 | 4 | 5
  dueAt: number                  // 下次到期时间
  interval: number               // 当前间隔（天）
  stability: number
  difficulty: number
  lapses: number                 // 累计遗忘次数
  lastGradedAt?: number
}

interface RecallLog {
  itemId: string
  sessionId: string              // 一局 = 一个学习会话
  responseType: 'teaching'       // 首次教学（不计分）
              | 'recognition4'   // 4 选 1
              | 'recognition-reverse' // 看义选词
              | 'spelling'       // 拼写
              | 'listening'      // 听音
  correct: boolean
  usedHint: boolean
  responseTime: number           // 毫秒
  graded: boolean                // 本次是否计入调度
  at: number
}
```

## 3. 学习闭环：先教，再考

测试之前必须有编码阶段。三阶段：

1. **首次遭遇（编码）**：新词首次出现时展示完整学习面板——释义、音标、发音、例句、词根。**不计错、不扣任何战斗资源**。玩家不会为尚未教过的内容受罚
2. **有提示回忆**：教学之后，当局内该词以低难度题型（4 选 1）提供过载，允许例句提示
3. **延迟独立回忆**：跨局、到期后，以高难度题型（拼写 / 听音）考核，这才是计分的主力

## 4. 战斗系统

### 4.1 基础框架

- 每回合 3 能量，摸 5 张牌（初始值，待平衡）
- 敌人有意图系统（telegraphed intent）

### 4.2 卡牌数值与掌握度脱钩（v2 核心修订）

v1 的「数值 = 基础值 × 掌握度系数」被否决：它诱导玩家偏爱熟词、逃避生词、囤积 L5 不肯毕业，过载从「可选」变成经济上的强制。修订为：

- **卡牌基础效果 = 固定基础值，与掌握度无关，打出永远全额生效**
- 掌握度只决定两件事：**过载的题型**（越熟题越难）和**过载的奖励倍率**（难题型配高奖励）
- 新词不比熟词弱，只是还没解锁过载资格

### 4.3 过载（Overcharge）修订版

- 打出单词牌时可选择「过载」：回答该词的回忆挑战
- **过载成功** → 奖励：效果加成或特效 + 语流 +1
- **过载失败** → **基础效果照常生效**，不哑火、不退能量；代价是语流清零 + SRS 记一次失败
- **L0 未教学词不提供过载**：首次打出触发教学面板（编码阶段），教学后当局内解锁 4 选 1 题型

**过载的代价（v2 新增，需评审确认）**：如果失败零代价，过载会变成每张牌的强制 quiz 税，节奏必死。因此引入「专注」资源：**每回合过载限 2 次**。稀缺性让「过载哪张牌」成为真决策——玩家自然挑有把握的词，等于自主安排复习强度；新词则安心按基础效果打出，不受惩罚。

### 4.4 题型分级（随掌握度提升）

| 掌握度 | 题型 | 性质 |
|---|---|---|
| L0 新词 | 教学面板 | 编码，不计分 |
| L1 认识 | 4 选 1（看词选释义） | 再认 |
| L2 熟悉 | 看释义选词 | 再认（反向） |
| L3 掌握 | 看义拼词 | 自由回忆 |
| L4 精通 | 听音辨词 | 自由回忆 |
| L5 毕业 | 退出日常牌组池 | 维护性复习 |

### 4.5 毕业与维护复习

- L5 词退出日常牌组池，转化为词币 + 「毕业墙」成就——毕业必须是奖励感，不是损失感
- 到维护复习日期（如 30 天后），毕业词作为「复习访客」重新注入候选池，验证长期保持

### 4.6 语流（Fluency）连击条

- 过载成功 +1 格；集满 3 格自动消耗，下回合 +1 能量
- 过载失败清零；上限默认 3 格（遗物可提升）

### 4.7 敌人

敌人机制带学习主题，例：「遗忘兽」的攻击暂时打乱玩家某张手牌的字母（制造提取难度）。数量见 MVP 范围。

## 5. SRS 计分规则（防刷题污染）

v1 的「每次答对升级、答错降级」被否决：同一场战斗反复出现会刷高掌握度，且混淆了再认与回忆。修订为：

1. **一局 = 一个学习会话（sessionId）。每个学习项目在每会话内只有第一次独立回忆计入调度（graded=true），后续重复只算练习**，不影响间隔（仍可给战斗奖励）
2. **评分按题型加权**：拼写 / 听音（自由回忆）> 看义选词 > 4 选 1（再认）。4 选 1 答对不能获得与独立拼写答对相同的进度
3. **使用提示（usedHint）降权**：提示后答对按更低权重计
4. **答对**：interval 按 stability 增长（基准 1 → 3 → 7 → 15 → 30 天，按题型权重微调），掌握度 +1
5. **答错**：lapses+1，interval ×0.5（**不清零**），掌握度 -1（不重置到底）
6. 教学面板（teaching）永远不计分

## 6. 节奏与无障碍（硬约束）

- **单场战斗目标 3-5 分钟**；完整 MVP 流程（2 普通战 + 1 Boss）10-15 分钟
- **每场战斗计分回忆（graded）上限 8 次**；每回合过载上限 2 次
- **软限时**：4 选 1 约 4 秒、看义选词 6 秒、拼写 20 秒、听音 15 秒。**超时不失败**，仅没有速度加成——绝不用倒计时制造失败
- **无障碍设置**：可关闭计时与动画、全键盘操作、听音题提供音标 / 音节拆分的文字替代形式

## 7. 内容管线

### 7.1 难度分级策略

- **内置内容包**：以人工分级为准——课程标签、考试大纲（初中 / 高中 / CET4 / CET6），后续可映射 CEFR。词频算法**不**用于内置包分级
- **自定义词表**：算法兜底排序（词频排名为主，词长 / 拼写不规则度修正），仅用于没有人工标签的场景

### 7.2 自定义导入协议

裸 `word[,definition]` 格式被否决（无法处理逗号、多义词、短语、重复、编码）。修订为导入向导：

1. **格式**：CSV / TSV，首行表头；提供模板下载
2. **列映射预览**：用户指定哪列是 word / definition / example，预览前 10 行确认
3. **标准化**：trim、大小写归一、Unicode NFC、全半角转换
4. **去重**：词形归一后去重，重复项列入报告
5. **错误报告**：行号 + 原因，可下载；允许「跳过错误行继续导入」
6. **来源标记**：导入项 `source='import'`，与内置、LLM 富化区分
7. **富化**：内置词典匹配释义 / 音标 / 例句；未命中词标记，LLM 富化后置

### 7.3 插画策略（v2 修订）

纯前端浏览器包不能安全保存图像生成服务密钥。修订为：

- **MVP**：不做 AI 插画，排版卡设计（无插画也必须好看）
- **完整版内置词库**：开发者离线批量生成（密钥在开发侧，不进包），打包为 assets
- **自定义词**：两条路——BYOK（用户自填密钥，仅存本地）或产品化阶段的后端代理。验证期不做

## 8. 技术架构

- **Vite + React + TypeScript**
- **纯 TS 领域核心**（`src/core/`）：战斗引擎 / SRS 调度器 / 词库与内容包 / 导入解析——与 UI 完全解耦，vitest 单测
- **UI 层**（`src/ui/`）：React + Zustand + framer-motion
- **数据**：内容包 JSON；SRS 状态与 RecallLog 存 IndexedDB；产品化后接后端
- **发音**：Web Speech API
- **目录约定**：`src/core/`（battle / srs / content / import）、`src/ui/`、`src/data/`、`tools/`（富化、插画脚本）、`docs/`
- **项目第一步**：先写 CLAUDE.md，定目录结构、命名约定、数据格式（约束先行）

## 9. MVP v0.1 范围（首个验证版本）

目标：用最小成本验证「过载好玩」+「学习有效」。**一场可重复战斗，不做爬塔**。

- 固定线性关卡：2 场普通战斗 + 1 场 Boss 战，可重复游玩
- 20-30 个学习项目（1 个内置级别包：CET4 高频 30 词）
- 2 种敌人 + 1 个 Boss、3 个遗物
- 新词教学面板 + 过载机制 + 基础 SRS（含会话内 graded 规则）
- 自定义导入（带列映射预览向导）
- **不做**：程序生成地图、商店、事件、AI 插画、语法内容、账号系统、云同步、移动端适配

### 成功标准（游戏性 + 学习性两组）

**游戏性**（自己 + 小范围试玩）：
- 战斗完成率 ≥ 70%
- 主动过载比例 ≥ 50%（过载是可选的，这个指标验证它是否有吸引力）
- 次日再次游玩率 ≥ 40%

**学习性**：
- 学前基线测试（确认词确实不会）
- 24 小时与 7 天后独立回忆率，**认义与拼写分开测**
- 目标：24h 认义 ≥ 70%、拼写 ≥ 50%；不接受只测 4 选 1 再认

## 10. 路线图（MVP 之后）

- **v0.2 爬塔结构**：程序地图、篝火（复习点）、商店、事件、完整遗物池
- **v0.3 内容扩展**：初中 / 高中 / 大学内容包分级上线、语法题型（选词填空 / 排序 / 改错 / 句转）
- **v1.0 产品化**：账号、云同步、内置词库插画、BYOK 或后端代理

## 11. 风险与待验证点

| 风险 | 应对 |
|---|---|
| 过载仍可能拖慢节奏 | 每回合限 2 次 + 软限时；MVP 首要验证手感 |
| 「专注」资源是否多余 | v0.1 试玩重点观察：限 2 次是制造决策还是制造烦躁 |
| 数值平衡 | 初始值拍脑袋，靠试玩迭代 |
| 词典数据版权 | 开源词频数据 + 自建释义，不用有版权词典 |
| 学习效果不可测 | 成功标准已含基线 + 24h/7d 独立回忆测试 |
