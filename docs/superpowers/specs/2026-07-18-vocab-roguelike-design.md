# 单词爬塔（Vocab Roguelike）设计文档

日期：2026-07-18
状态：已确认，待写实现计划

## 1. 产品定位

一款学习英语单词的卡牌 Roguelike 游戏，玩法框架参考杀戮尖塔（Slay the Spire）。

- **目标用户**：中文英语学习者（产品化方向，非自用工具）
- **平台**：Web 浏览器（验证期），后续可包壳
- **定位**：**游戏为体，学习为主题**。先保证好玩、留得住人；学习效果通过机制设计内生获得，不靠强制测验
- **核心信条**：「变强」和「学会单词」是同一件事，不是交易

## 2. 核心概念映射

| 杀戮尖塔 | 本作 |
|---|---|
| 卡牌 | 一个单词 |
| 出牌 | 可选择「过载」：快速回忆该词以强化效果 |
| 卡牌数值 | 基础值 × SRS 掌握度系数 |
| 构筑牌组 | 选择本局带哪些词（复习词 + 新词） |
| 卡牌升级 | 掌握度升级：认识 → 熟悉 → 掌握 → 精通 |
| 删牌 | 单词毕业（L5），移出牌组换词币 |
| 遗物 | 学习科学主题增益（词根、遗忘曲线等） |
| 篝火 / Boss | 复习点 / 阶段大考 |
| 死亡惩罚 | 答错的词下局优先出现（遗忘曲线驱动） |

## 3. 核心循环（一局）

1. **开局组牌**：SRS 调度器给出候选词池（今日到期复习词 + 新词配额），玩家挑出本局牌组（默认 12 张：8 复习 + 4 新词，比例可调）
2. **爬塔**：程序生成的分支地图，节点类型 = 战斗 / 精英 / 篝火 / 商店 / 事件 / Boss
3. **战斗**：回合制卡牌，每回合做「过载」抉择
4. **战后奖励**：三选一奖励卡（新词入组），本场回忆数据写入 SRS
5. **死亡或通关**：本局所有单词掌握度变化落盘，解锁后续章节

## 4. 战斗系统

### 4.1 基础框架

- 每回合 3 能量，摸 5 张牌（STS 经典数值框架，初始值，待平衡）
- 敌人有意图系统（telegraphed intent）

### 4.2 单词牌

- 牌面：单词 + 词性 + 插画（插画缺失时用排版卡兜底）
- 效果类型：攻击 / 格挡 / 抽牌 / 回复等
- 数值 = 基础值 × 掌握度系数（初始系数：L0=0.6 / L1=0.8 / L2=1.0 / L3=1.2 / L4=1.5 / L5=1.5，待平衡）

### 4.3 过载（Overcharge）——核心机制

打出每张单词牌时二选一：

- **直接打**：基础效果。低掌握度的词数值很低
- **过载回忆**：弹出 1-2 秒快速回忆挑战
  - 答对 → 效果 ×1.5~2 或附加特效（吸血 / 多抽牌等）
  - 答错 → 卡牌哑火，能量不退还

设计要点：回忆是**可选的风险抉择**，不是强制税。新词基础数值低，玩家天然想过载它；「我想记住这个词」成为自发的变强动机。

### 4.4 回忆形式分级（随掌握度提升变难）

| 掌握度 | 回忆形式 |
|---|---|
| L0-L1 新词/认识 | 4 选 1（看词选释义） |
| L2 熟悉 | 看释义选词 |
| L3 掌握 | 看义拼词（拼写） |
| L4 精通 | 听音辨词（Web Speech API 发音） |
| L5 毕业 | 免测，直接打出满效果；可移出牌组换词币 |

### 4.5 语流（Fluency）连击条

- 连续过载成功 +1 格；集满 3 格后自动消耗，下回合 +1 能量
- 过载答错 → 清零
- 语流上限默认 3 格（遗物可提升）
- 滚雪球机制，奖励真实的流利度

### 4.6 敌人

- MVP：6 种普通敌人 + 1 个 Boss
- 敌人机制带学习主题，例：「遗忘兽」的攻击会暂时打乱玩家某张手牌的字母（制造记忆提取难度）

## 5. SRS 系统（后台隐形运转）

### 5.1 掌握度模型

- L0-L5 六级：新词 → 认识 → 熟悉 → 掌握 → 精通 → 毕业
- 每次回忆（过载尝试、篝火复习、Boss 考核）都是一次评分：答对升级，答错降级
- 简化间隔重复（SM-2 思想）：答对后下次复习间隔延长（1 → 3 → 7 → 15 → 30 天），答错重置

### 5.2 调度器

- 按遗忘曲线计算「今日到期词」，优先注入下一局的候选词池
- 玩家感知到的是「这局牌组正好有我不熟的词」，感知不到算法

### 5.3 词库 = 章节

- v1 内置 CET4 高频词库（用户群最大）
- 完整版每幕约 300 词池，通关解锁下一幕；MVP 只做 1 幕，词池 50 词（见第 9 节）
- 词卡数据字段：`word, phonetic, pos, definitions(zh), example, exampleZh, roots, frequencyRank, difficultyScore, act`

## 6. 元游戏

- **篝火**：复习点——快速过一遍本局答错的词，答对回血（学习反馈即时化）
- **商店**：词币买新词卡 / 移除毕业词换资源 / 买遗物
- **遗物**（学习科学主题，MVP 8 个）：
  - 词根词典：同词根单词互相 +1 伤害
  - 艾宾浩斯沙漏：每局首次答错不降级
  - 语流护符：语流上限 +2
  - 例句卷轴：过载时显示例句提示
  - 同义词网络：连续打出同义词时伤害递增
  - 词币磁铁：词币掉落 +50%
  - 复读机：每回合第一次过载答错不清语流
  - 错题本：篝火复习量翻倍
- **事件**（学习主题随机事件）：
  - 同义词陷阱：辨认同义词拿奖励，认错扣血
  - 生词宝箱：直接获得 3 张新词卡，但下两场战斗敌人增强
  - 拼写擂台：限时拼写挑战换遗物

## 7. 内容管线

### 7.1 词库管线

内置词库之外，支持自定义词表导入：

1. **导入**：CSV 文件或粘贴文本（格式：`word[,definition]` 每行一条）
2. **自动富化**：内置词典数据匹配释义、音标、例句；词频表（COCA 词频排名）计算难度分
3. **难度模型**：`difficultyScore = f(词频排名, 词长, 拼写不规则度)`
4. **分幕**：按难度分排序，每幕约 300 词池，自动生成关卡曲线，无需人工编排
5. **未收录词**：标记出来，走 LLM 富化（生成释义例句）兜底（MVP 后置）

### 7.2 插画管线

卡面插画用 AI 图像生成（双重编码效应：图像 + 文字同时编码，记忆留存显著高于纯文字——插画是学习加成，不是美术负担）。

- 统一风格提示词模板，保证全套卡面风格一致
- **内置词库**：离线批量生成，打包进 assets（脚本放 `tools/`，可用 codex 批量跑）
- **自定义词**：运行时按需生成 + IndexedDB 缓存，生成完成前用排版卡兜底
- **成本控制**：只为玩家实际遇到的词生成，不预生成全库

## 8. 技术架构

- **Vite + React + TypeScript**
- **纯 TS 领域核心**（`src/core/`）：战斗引擎 / SRS 调度器 / 地图生成 / 词库——与 UI 完全解耦，可单测（vitest），未来可移植
- **UI 层**（`src/ui/`）：React + Zustand 状态 + framer-motion 卡牌动画
- **数据**：词库 JSON 打包；玩家进度存 localStorage/IndexedDB；产品化后再接后端账号体系
- **发音**：Web Speech API（零成本内置 TTS）
- **目录约定**：
  - `src/core/` 纯逻辑（battle / srs / map / wordlist）
  - `src/ui/` 组件与界面
  - `src/data/` 词库 JSON
  - `tools/` 词库富化、插画生成脚本
  - `docs/` 文档
- **项目第一步**：先写 CLAUDE.md，定目录结构、命名约定、词库数据格式（约束先行）

## 9. MVP 范围

验证两个假设：**过载机制好玩** + **打完一局真记住了词**。

- 1 幕地图（15 节点）、1 个角色、50 张单词卡（取自 CET4 高频词表，即 MVP 词池 50 词）
- 6 种敌人 + 1 个 Boss、8 个遗物、篝火 / 商店 / 事件各一
- 自定义导入：CSV + 内置词典富化（LLM 富化后置）
- 卡牌：排版优先设计（无插画也必须好看），内置池批量插画作为打磨项
- **不做**：账号系统、云同步、多词库切换 UI、移动端适配、LLM 富化
- **成功标准**：自己愿意玩 10 局以上；局后 24 小时单词再认率 ≥ 70%（可量化）

## 10. 风险与待验证点

| 风险 | 应对 |
|---|---|
| 过载打断战斗节奏 | 回忆挑战限时 1-2 秒；MVP 首要验证手感 |
| 数值平衡（掌握度系数、过载倍率） | 初始值拍脑袋，靠试玩迭代 |
| 词典数据版权 | 用词频开源数据 + 自建释义，不用有版权词典 |
| 插画风格不一致 | 统一提示词模板 + 固定 seed 策略 |
| 图像生成成本 | 按需生成 + 缓存，不预生成全库 |
