# 英语爬塔 MVP 试玩与学习保持测试协议

版本：v0.1 候选版
目标人数：至少 10 名目标学习者
单次时长：首次 25–35 分钟，其中游戏会话目标为 10–15 分钟

## 1. 研究目标

验证两个问题：

1. 玩家是否愿意在卡牌决策中主动使用英语回忆，而不是把练习视为阻断游戏的弹窗。
2. 一次 10–15 分钟会话能否产生可观察的 24 小时与 7 天词义、拼写保持信号。

本轮不以小样本证明长期学习效果，也不根据单次战斗成绩给参与者贴等级。

## 2. 参与者与分组

- 至少 10 人，覆盖初中、高中、大学三个学习阶段；每个阶段尽量不少于 3 人。
- 参与者最近 30 天内有英语学习需求，并能独立使用桌面浏览器。
- 记录自报英语阶段、每周学习频率、卡牌游戏经验；不收集真实姓名，使用随机参与者 ID。
- 如使用同一批目标词，事先排除全部已熟练掌握或完全超出水平的参与者。

## 3. 知情说明与数据边界

开始前明确说明：

- 试玩记录用于改进游戏和学习流程，可随时退出。
- MVP 的详细数据保存在本地；研究员只取得参与者主动导出的文件和问卷答案。
- 诊断事件不包含自由输入答案、导入原文、姓名、邮箱或设备账号。
- 错误与未答对不会带来额外降级、生命惩罚或羞辱性标签。

## 4. 材料准备

- 固定构建版本和内容包版本。
- 10–12 个目标词义：4 个低熟悉、4 个中等熟悉、2–4 个干扰项。
- 基线题、24 小时复测、7 天复测采用等价但不重复展示答案的独立题目。
- 屏幕录制仅在参与者单独同意后启用；录制中不得出现其他个人应用。
- 准备 Chrome、Edge、Safari 当前稳定版的冒烟检查表。

## 5. 流程

### A. 基线（5–8 分钟）

不显示答案，完成：

- 目标词认义选择。
- 根据中文义拼写英文。
- 自报每个词的熟悉度：陌生 / 见过 / 基本会 / 熟练。

### B. 首次游戏（10–15 分钟）

参与者独立完成：

1. 选择内容包与新词数量。
2. 完成战前教学。
3. 完成两场普通战、奖励和 Boss 战；允许失败，不给予操作提示，只有卡住 60 秒后才询问其意图。
4. 查看结算。
5. 可选任务：导入一份 4–10 行自定义 CSV 并开始会话。

观察员只记录事实：退出位置、误触、未理解术语、是否主动过载、是否机械耗尽专注、是否能说明敌人意图。

### C. 即时访谈（8–12 分钟）

每题 1–7 分并追问原因：

- 战斗本身有趣。
- 英语练习与战斗结合自然。
- 过载是有意义的选择。
- 答错后的反馈让我愿意继续。
- 我理解专注、语流、勇气奖励和敌人意图。
- 我愿意明天再玩一局。

开放问题：最想保留、最想删除、最困惑、最希望新增的内容。

### D. 24 小时复测（5–8 分钟）

- 不先打开游戏，不显示答案。
- 完成等价认义与拼写题。
- 记录是否在 24–48 小时内主动开始第二次有效会话。

参考目标：认义正确率 ≥ 70%，拼写正确率 ≥ 50%，次日有效会话率 ≥ 40%。

### E. 7 天复测（5–8 分钟）

- 使用与 24 小时口径一致的独立题目。
- 记录保持趋势，不设置小样本强制上线门槛。
- 若相对 24 小时大幅下降，优先复查调度与教学，不增加惩罚。

## 6. 指标计算

| 指标 | 口径 |
|---|---|
| 会话完成率 | 到达胜利结算的会话 / 已开始会话 |
| 退出位置 | `session_abandoned` 的阶段分布 |
| 专注使用率 | 已完成过载次数 / 已进入战斗可用专注总数 |
| 到期覆盖率 | 尝试过的到期项目数 / 本局调度的到期项目数 |
| 直接/提示/错误 | 直接答对、提示后答对、未答对的事实计数 |
| 24h / 7d 保持 | 独立复测正确数 / 独立复测目标数 |
| 次日再次游玩率 | 24–48 小时内再次开始有效会话人数 / 可回访人数 |

同一到期窗口的重复作答只允许第一次合格作答改变 SRS，但所有尝试可作为非计分诊断事实保留。

## 7. 平衡记录

每位参与者记录：

- 单战与整局时长。
- 每战剩余生命、专注使用、过载正确/提示/错误。
- 是否出现明显必败、无脑必赢或必须机械耗尽专注的策略。
- 如果玩家总是耗尽专注，先记录为实验结果，不临时增加惩罚。
- 200% 缩放、键盘路径、减少动画、语音不可用降级是否可完成。

## 9. 缺陷与决策规则

- P0：数据损坏、无法启动、SRS 可重复刷分、隐私原文进入诊断或主要流程完全阻塞。
- P1：常规路径无法完成一局、导入后不能开局、键盘或语音降级阻塞。
- P2：局部文案、视觉或非阻塞反馈问题。
- P0/P1 必须在候选发布前关闭；玩法参数调整必须引用试玩记录，不在单次观察后临时改变学习规则。
